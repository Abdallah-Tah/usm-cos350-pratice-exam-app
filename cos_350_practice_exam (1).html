Grep Quick Rules

^ start of line, $ end of line, . any one character, [abc] any one listed char, [^abc] any one char not listed, * zero or more, + one or more in egrep, ? zero or one, {n} exactly n in egrep.

Permission Mask Quick Rules

The bottom 9 bits of st_mode are rwxrwxrwx. User bits are the top three, group bits are the middle three, and other bits are the bottom three. Example masks: S_IROTH = 00004, S_IWOTH = 00002, S_IXOTH = 00001.

Set 1: UNIX Basics

Rule to remember: many short UNIX questions ask what a command does, what option changes its behavior, or what exact command you should type.
Question 1

How do you view the manual page for the UNIX command grep? Explain what the command does, not only the text you type.

Show answer

man grep

This opens the manual page for the grep command.

Question 2

You created a file named notes.txt. Set the permissions so that everyone can read it, but only you can write it. Explain both the symbolic and numeric meaning.

Write one symbolic mode answer and one numeric mode answer.
Show answer

Symbolic:

chmod u+w,go-w notes.txt
chmod a+r notes.txt

Or more simply:

chmod 644 notes.txt

644 means user = read/write, group = read, others = read.

Show answer

It prints all lines in words that start with un.

The ^ means beginning of the line.

Question 4

Assume words is a file with one word per line. What does this command do? Explain what each grep part means.

Show answer

It prints all words that end with ing.

The $ means end of the line.

Set 2: Systems Programming Basics

Question 6

What system call can you use to get information about a file, including its size in bytes?

Show answer

stat()

The file size is stored in the st_size field of the struct stat.

Question 7

Which function is commonly used for buffered file input, and which one is used for unbuffered file input?

Show answer

Buffered input: fread(), fgets(), or getc() depending on the situation.

Unbuffered input: read()

In many class questions, the expected pair is fread() for buffered and read() for unbuffered.

Question 8

What is the difference between open() and fopen()?

Show answer

open() is a UNIX system call and returns a file descriptor (an int).

fopen() is a C library function and returns a FILE *.

fopen() uses buffered I/O. open() is lower level.

Question 9

Explain the parameters of qsort():

qsort(base, nmemb, size, compar);
Show answer

base: pointer to the array

nmemb: number of elements in the array

size: size in bytes of each element

compar: pointer to the comparison function used to decide order

qsort() can sort any kind of array because you give it the start address, number of items, item size, and a comparison function.

Question 10

Why must the comparison function passed to qsort() accept const void * parameters?

Show answer

Because qsort() is a generic function. It does not know the real type of the array elements.

It passes element addresses as const void *, and inside the comparison function you cast them to the correct type.

Set 3: Permissions and Bit Masks

Rule to remember: permission masks are usually written in octal. The order is user, group, other. Each group is r=4, w=2, x=1.
Question 11

The bottom 9 bits of st_mode store file permissions in the order rwxrwxrwx. What numeric mask represents the other write permission bit? Explain how you know its position.

Show answer

0002 in octal

In hex, that is 0x2.

Question 12

Write an if statement that prints "group can execute" if the group execute bit is set. Explain why a bitwise AND is used.

struct stat info;
stat("file", &info);
Show answer 
if (info.st_mode & 0010) {
    printf("group can execute\n");
}

The group execute bit is octal 0010.

Question 13

Write an if statement that prints "everyone readable" if user, group, and others all have read permission. Explain why the comparison checks all three read bits together.

Show answer 
if ((info.st_mode & 0444) == 0444) {
    printf("everyone readable\n");
}

0444 checks all three read bits together.

Set 4: C Language Questions

Question 14

C traditionally does not require a built-in Boolean type. How are true and false usually represented in C?

Show answer

False is represented by 0.

True is represented by any nonzero value, such as 1, 2, or -5.

Question 15

Fix this function so it changes the caller's variable:

void add5(int x) {
    x = x + 5;
}
Show answer 
void add5(int *x) {
    *x = *x + 5;
}

int main(void) {
    int a = 10;
    add5(&a);
    return 0;
}

You must pass the address and use a pointer to change the original variable.

Question 16

Write your own version of strlen() without using any other library function.

Show answer 
int my_strlen(char *s) {
    int len = 0;
    while (*s != '\0') {
        len++;
        s++;
    }
    return len;
}
Question 17

What is wrong with this code?

char *p;
*p = 'A';
Show answer

p does not point to valid memory yet. Dereferencing it is invalid.

You must make it point to allocated memory or to an existing variable first.

char c;
char *p = &c;
*p = 'A';
Question 18

What is the output of this code?

int x = 7;
int *p = &x;
printf("%d\n", *p);
Show answer

The output is 7.

p stores the address of x, and *p means the value stored at that address.

Set 6: Experiment-Style Practice (Exp 1 to Exp 12)

These are based on the kind of work you have been practicing: shell commands, grep, permissions, arrays, debugging, recursion, file input/output, links, and compression-style thinking.

Question 25

You run this command:

ls -R

What does it do? Does it always go into an infinite loop if symbolic links exist?

Show answer

ls -R lists directories recursively, showing files and subdirectories under the current directory.

No, it does not always go into an infinite loop. Normal ls -R lists directory contents recursively, but it does not blindly follow symbolic links as if they were normal directories in a way that causes endless recursion.

Question 26

What is the difference between a hard link and a symbolic link?

Show answer

A hard link is another directory entry for the same inode and same file data.

A symbolic link is a separate file that stores a pathname to another file.

If the original file is removed, a hard link can still access the data, but a symbolic link may become broken.

Question 27

Write the command to create a symbolic link named words_link that points to the file /usr/class/cos350/exps/l7/words.

Show answer 
ln -s /usr/class/cos350/exps/l7/words words_link
Question 28

Using grep on a word list, how would you count:

  1. words containing cie
  2. words containing ei not immediately preceded by c
  3. words starting with ei
Show answer 
grep 'cie' words | wc -l
grep '[^c]ei' words | wc -l
grep '^ei' words | wc -l
Question 29

What is wrong with this kind of array code?

int a[10];
a[10] = 5;
Show answer

The valid indexes are 0 through 9. Writing to a[10] is out of bounds.

This causes undefined behavior.

Question 30

Why must a variable used with getchar() usually be declared as int and not char?

Show answer

Because getchar() must be able to store every valid character value and also the special value EOF.

char may not be able to represent EOF correctly, but int can.

Question 31

Suppose you are writing your own version of expand. What should happen when you read a tab character?

Show answer

You should output enough spaces to move to the next tab stop.

The number of spaces depends on the current column position and the tab width.

Question 32

In GDB, what commands would you use to:

  1. set a breakpoint at line 26
  2. run the program
  3. print the value of variable local
  4. print the address of local
Show answer

run
p local
p &local
Question 33

What does recursion mean in C programming?

Show answer

Recursion means a function calls itself.

A recursive function should have a base case to stop the recursion, otherwise it can continue until stack overflow.

Question 34

Why is a base case important in a recursive function?

Show answer

The base case is the stopping condition.

Without it, the function keeps calling itself and may crash from too many recursive calls.

Question 35

What is the difference between these two declarations?

char *argv[]
char **argv
Show answer

They are closely related and in function parameters they are effectively equivalent.

Both represent a pointer to pointers to char, commonly used for command-line arguments.

Set 7: Project-Style C Questions

Question 36

When using open(), what value usually means the open failed?

Show answer

A return value less than 0, usually -1, means failure.

Question 37

Why should you check the return value of read()?

Show answer

Because read() tells you how many bytes were actually read.

A value of 0 means end of file, and a negative value means an error.

Question 38

In a compression project, why might you keep a bit buffer and a count of how many bits are currently stored?

Show answer

Because compressed data may not align exactly to full bytes.

A bit buffer lets you pack bits together until you have enough to write one byte to the output file.

Question 39

Why is it useful to split a large program into helper functions instead of writing everything in main()?

Show answer

It makes the code easier to read, test, debug, and reuse.

It also keeps main() shorter and easier to understand.

Question 40

What is the difference between text mode thinking and byte-level file processing thinking in systems programming?

Show answer

Text mode thinking focuses on characters, lines, and strings.

Byte-level processing focuses on raw bytes, exact sizes, file descriptors, and how data is stored and moved.

Set 10: Devices, Terminals, and Signals

Question 55

Unix treats devices like files. What does that mean for programming with devices?

Show answer

It means many devices can be used with the same basic file operations such as open(), close(), read(), and write().

This is one of the main Unix ideas: regular files and many devices use a similar interface.

Question 56

What command shows current terminal settings?

Show answer

stty -a

This prints terminal settings such as special control characters, echo behavior, and canonical mode related information.

Question 57

What do tcgetattr() and tcsetattr() do?

Show answer

tcgetattr() reads terminal attributes into a struct termios.

tcsetattr() changes terminal attributes using values from a struct termios.

They are used to control terminal behavior such as echo and input buffering.

Question 58

Why would a password program turn off terminal echo?

Show answer

So the password is not shown on the screen while the user types it.

This is done by changing terminal settings, usually by clearing the echo bit in the local terminal flags.

Question 59

What does fcntl(fd, F_GETFL) do?

Show answer

It reads the file status flags of the open file descriptor fd.

The result can then be modified and written back with F_SETFL.

Question 60

Why is append mode useful when multiple programs write to the same log file?

Show answer

Append mode helps each write go to the end of the file.

Without append mode, two programs may both seek to the end and then write in a conflicting order.

Question 61

What does the command kill 1234 do?

Show answer

It sends the default signal, usually SIGTERM, to process 1234.

This asks the process to terminate.

Question 62

What does the command kill -9 1234 do?

Show answer

It sends SIGKILL to process 1234.

This forcefully kills the process and cannot be caught or ignored.

Question 63

What signal is normally sent by pressing Ctrl-C? What signal is normally sent by pressing Ctrl-Z?

Show answer

Ctrl-C sends SIGINT.

Ctrl-Z sends SIGTSTP.

Question 64

What does this line do?

signal(SIGINT, handler);
Show answer

It installs handler as the signal handler for SIGINT.

When the program receives SIGINT, the handler function runs.

Question 65

What does this line do?

signal(SIGINT, SIG_IGN);
Show answer

It tells the program to ignore SIGINT.

That means pressing Ctrl-C will not use the normal interrupt behavior for that process.

Question 66

What does sleep(5) do?

Show answer

It pauses the calling process for up to 5 seconds.

If a non-ignored signal arrives first, the sleep can end early.

Question 67

What does alarm(4) do?

Show answer

It schedules a SIGALRM signal to be sent to the calling process in 4 seconds.

Question 68

What does pause() do?

Show answer

It stops the calling process until a signal is delivered.

If a signal-catching function runs and returns, pause() returns -1 and errno is set to EINTR.

Question 69

Explain what happens in this kind of program:

signal(SIGALRM, wakeup);
alarm(4);
pause();
Show answer

First the program installs a handler for SIGALRM. Then it schedules an alarm for 4 seconds. Then it blocks in pause().

When the alarm expires, the kernel sends SIGALRM, the handler runs, and then the program continues after pause().

Set 9: Arrays, Strings, and Command Arguments

Question 46

In C, if you declare:

int a[10];

What are the valid indexes of the array?

Show answer

The valid indexes are 0 through 9.

C arrays start at index 0 and go to size-1.

Question 47

Why is this dangerous in C?

a[10] = 0;
Show answer

This writes outside the array bounds.

C does not perform automatic bounds checking, so this causes undefined behavior.

Question 48

What does the array name represent in memory?

int a[10];
Show answer

The array name represents the starting address of the array in memory.

For example, a is the address of a[0].

Question 49

Why must C strings always include space for one extra character?

Show answer

Because C strings end with the null terminator '\0'.

This character marks the end of the string.

Question 50

Why can this code cause a buffer overflow?

char buf[80];
scanf("%s", buf);
Show answer

scanf does not know the size of the buffer.

If the user types more than 79 characters, it will write past the array.

Question 51

How can you fix the previous code to avoid overflow?

Show answer 
scanf("%79s", buf);

This limits the input length.

Question 52

What does this declaration mean?

int main(int argc, char *argv[])
Show answer

argc is the number of command-line arguments.

argv is an array of strings containing the arguments.

Question 53

If the command is:

myexpand -t 4

What are the values of argc and argv?

Show answer

argc = 3

argv[0] = "myexpand"

argv[1] = "-t"

argv[2] = "4"
